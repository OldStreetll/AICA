# 代码重构流程

当用户要求重构代码时，请按以下流程操作：

## 1. 分析现状（Analyze）

- 理解当前代码的功能和行为
- 识别要改善的问题（重复代码、过长方法、紧耦合等）
- 确认重构范围——只改用户要求的部分

## 2. 规划（Plan）

- 确定重构步骤的顺序
- 评估每步的影响范围
- C/C++：检查公共 API 是否会变化，如果变化需要同步头文件和所有调用者

## 3. 重构（Refactor）

- 每次只做一个小改动
- 保持行为不变（这是重构的核心约束）
- 不添加新功能，不修复无关 Bug
- 方法提取、重命名、移动等操作应保持语义等价

## 4. 验证（Verify）

- 确认重构后行为与重构前完全一致
- 检查编译无新增错误或警告
- 检查所有调用者是否仍然正常

## 成功标准

- [ ] 重构后行为与重构前完全一致
- [ ] 修改范围不超出用户请求
- [ ] 公共 API 变化已同步到头文件和调用者（如适用）
- [ ] 编译无新增错误

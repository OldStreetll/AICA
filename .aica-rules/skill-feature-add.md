# 新功能开发流程

当用户要求添加新功能或新代码时，请按以下流程操作：

## 1. 理解需求（Understand）

- 明确用户要实现什么功能
- 确认功能边界（做什么、不做什么）
- 如果需求不清晰，先询问用户

## 2. 设计（Design）

- 确定修改哪些文件
- 考虑与现有代码的集成点
- 如果涉及新类/接口，先规划结构
- C/C++ 项目：考虑头文件声明和源文件实现的分离

## 3. 实现（Implement）

- 按设计逐步实现
- 遵循项目现有的代码风格和命名规范
- 新增代码需要适当的注释
- 处理边界情况和错误路径

## 4. 集成（Integrate）

- 确认新代码与现有代码正确集成
- 检查编译是否通过
- 如果有头文件变化，确认声明和实现一致

## 成功标准

- [ ] 功能完整实现用户需求
- [ ] 代码风格与项目一致
- [ ] 头文件和源文件保持同步（如适用）
- [ ] 编译无新增错误

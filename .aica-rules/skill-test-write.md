# 测试编写流程

当用户要求编写测试时，请按以下流程操作：

## 1. 分析被测代码（Analyze）

- 阅读并理解被测函数/类的行为
- 识别关键路径、边界条件、异常路径
- 了解被测代码的依赖项

## 2. 设计测试用例（Design）

- 覆盖正常路径（happy path）
- 覆盖边界条件（空输入、极值、边界长度等）
- 覆盖异常路径（无效输入、异常抛出等）
- 使用项目现有的测试框架和断言风格

## 3. 编写测试（Write）

- 遵循 Arrange-Act-Assert 模式
- 每个测试方法测试一个行为
- 测试方法命名：MethodName_Condition_ExpectedResult
- 使用 Mock 或 Stub 隔离外部依赖
- 参考项目中现有测试的风格

## 4. 验证（Verify）

- 确认测试编译通过
- 确认测试命名清晰表达意图
- 确认没有遗漏关键的测试场景

## 成功标准

- [ ] 覆盖正常路径、边界条件和异常路径
- [ ] 遵循项目现有的测试风格
- [ ] 测试编译通过
- [ ] 每个测试独立可运行，不依赖执行顺序

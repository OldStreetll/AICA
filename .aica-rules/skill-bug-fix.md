# Bug 修复流程

当用户报告了一个Bug或要求修复问题时，请按以下流程操作：

## 1. 诊断（Diagnose）

- 理解用户描述的症状
- 复现问题：阅读相关代码，理解当前行为
- 如果有错误信息，分析错误栈和错误类型

## 2. 定位（Locate）

- 使用 grep/read_file 找到相关代码
- 从错误信息追溯到具体代码行
- 确认根因（root cause），而非表面症状
- 检查是否有多处需要修改

## 3. 修复（Fix）

- 只修改必要的代码，不做无关的重构或美化
- 修复应该最小化影响范围
- 如果修改涉及头文件签名变化（C/C++），同步修改对应的 .h/.hpp 文件

## 4. 验证（Verify）

- 确认修改后的逻辑正确
- 检查是否引入新问题
- 如果有相关测试，确认测试仍通过

## 成功标准

- [ ] 根因已明确识别并记录
- [ ] 修复是最小化的（不包含无关改动）
- [ ] 头文件/源文件保持同步（如适用）
- [ ] 没有引入新的编译错误或警告

# AICA 全功能手动测试计划 — 基于 microsoft/calculator 项目

> 版本: 1.0
> 日期: 2026-03-16
> 测试目标项目: D:\project\calculator (microsoft/calculator)
> 测试对象: AICA Visual Studio 2022 Extension (全部 14 个工具 + Agent 循环 + 响应质量)

---

## 一、测试目标

使用 microsoft/calculator 作为真实 C++ 项目，手动验证 AICA 在以下三个维度的表现：

| 维度 | 定义 |
|------|------|
| **正确性** | AICA 的输出与项目实际内容一致，无事实错误、无虚构内容 |
| **准确性** | AICA 对代码结构、逻辑、依赖关系的理解精确，定位准确 |
| **完整性** | AICA 的回答覆盖了用户问题的所有关键方面，无重要遗漏 |

---

## 二、测试环境准备

1. 使用 `build.ps1` 编译 AICA 解决方案，确认 0 error
2. 在 Visual Studio 2022 中 F5 启动实验实例
3. 在实验实例中打开 `D:\project\calculator\src\Calculator.sln`
4. 打开 AICA 聊天窗口
5. 打开 VS 的 **Output → Debug** 窗口，筛选 `[AICA]` 前缀日志
6. 确保已配置 LLM API 连接（环境变量 `OPENAI_API_KEY` 或 Options 中设置）

---

## 六、已知注意事项

1. **calculator 项目构建需求**: 需要 UWP 开发工作负载和 Windows 11 SDK，部分测试（如 TC-A11 构建命令）可能需要完整构建环境
2. **路径中含空格**: `Header Files/` 目录名包含空格，可测试 AICA 对特殊路径的处理能力
3. **C++/CX 语法**: calculator 使用了 C++/CX 扩展语法（`ref class`、`^` 指针），可检验 AICA 对非标准 C++ 的理解
4. **混合语言**: 解决方案同时包含 C++ 和 C# 项目，可测试跨语言分析能力
5. **与 ResponseQuality_ManualTestPlan.md 的关系**: 本计划的 H 节与已有响应质量测试计划互补，H 节聚焦于 calculator 项目场景，已有计划聚焦于通用响应质量

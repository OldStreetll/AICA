  AICA 工具调用功能人工测试指南

  🚀 第一步：启动测试环境

  1.1 启动 Experimental Instance

  1. 在 VS 中打开：D:\Project\AIConsProject\AIHelper\src\AICA.sln
  2. 右键 "AICA.VSIX" 项目 → "设为启动项目"
  3. 按 F5 启动 Experimental Instance
  4. 等待 VS 完全加载

  1.2 验证扩展加载

  在 Experimental VS 中：
  - 菜单：扩展 → 管理扩展
  - 确认 "AICA" 显示为已安装并启用

  1.3 打开聊天窗口

  方法1（推荐）：按快捷键 Ctrl+Alt+A
  方法2：菜单 → 视图 → 其他窗口 → AICA Chat
  方法3：在代码编辑器右键 → AICA → Open AICA Chat

  🔧 第二步：配置 API（首次使用）

  2.1 设置 API 端点

  1. 菜单：工具 → 选项 → AICA → General
  2. 设置 API Endpoint（例如：https://api.openai.com/v1）
  3. 设置 API Key 和其他必要参数
  4. 确保 "Enable Tool Calling" 已勾选
  5. 点击 "确定"

  📋 第三步：基础功能测试

  3.1 测试搜索工具（GrepSearchTool）

  输入消息：
  搜索项目中包含 "ToolDispatcher" 的文件

  预期结果：
  - ✅ 返回包含该关键词的文件列表
  - ✅ 响应时间合理（几秒内）
  - ✅ 不出现权限审批弹窗

  如果失败：检查项目是否正确加载，文件是否存在

  3.2 测试文件查找工具（FindByNameTool）

  输入消息：
  找到名为 "ToolExecutionPipeline.cs" 的文件

  预期结果：
  - ✅ 返回正确的文件路径
  - ✅ 执行流畅无异常

  3.3 测试文件读取工具（ReadFileTool）

  输入消息：
  读取 ToolExecutionPipeline.cs 文件的前50行

  预期结果：
  - ✅ 显示文件内容
  - ✅ 格式正确，代码高亮

  🔐 第四步：权限控制测试

  4.1 测试需要审批的工具（AttemptCompletionTool）

  输入消息：
  请完成当前任务并给出最终总结

  预期结果：
  - ✅ 应该触发权限审批对话框（如果 PermissionCheckMiddleware 已接入）
  - ✅ 选择"拒绝"：返回拒绝消息，不继续执行
  - ✅ 选择"批准"：继续执行并给出总结

  如果没有审批弹窗：说明权限中间件可能未完全接入，这是正常的（取决于具体实现）

  4.2 测试追问工具（AskFollowupQuestionTool）

  输入消息：
  你需要向我确认一些细节才能继续工作

  预期结果：
  - ✅ 同样应触发审批流程（如果已实现）
  - ✅ 审批后能正常提出追问

  🐛 常见问题排查

  如果工具调用失败：

  1. 检查 API 配置是否正确
  2. 查看输出窗口的错误日志
  3. 确认 "Enable Tool Calling" 已启用
  4. 重启 Experimental Instance

  如果权限控制不生效：

  1. 检查是否正确注册了 PermissionCheckMiddleware
  2. 确认 IPermissionHandler 实现是否正确
  3. 查看中间件执行顺序

  如果性能异常：

  1. 检查超时设置是否合理
  2. 观察内存使用情况
  3. 查看是否有资源泄露

  按照这个测试流程，你可以全面验证 AICA 工具调用系统的功能完整性和安全性。
